# ダルライザー THE GAME 引き継ぎ資料 v2

## プロジェクト概要

ダブルドラゴン（スーパーファミコン）風のベルトスクロールアクションゲーム。
ブラウザ上で動作する単一HTMLファイル（約1.5〜2MB）。
全キャラクタースプライトはGemini AIで生成し、グリーンバック除去して組み込み済み。

## 最新バージョン

**game_v8b.html** が最新版（v8のフレームマッピング修正版）。

## 🚨 現状の問題点（最優先で対応が必要）

### 問題1: スプライト切り出しの重複

**症状:** ダルライザーのモーションが不自然。パンチ・キック・ガード時に隣のポーズの手足が映り込む。

**原因:** Geminiが生成したスプライトシートはポーズ間の間隔が狭く、等幅分割すると隣接するスプライトの一部が含まれてしまう。1つ1つ画像を切り出しているのではなく、行を均等分割した四角い範囲をそのまま表示しているため。

**解決策（2つ）:**

A) **Geminiに間隔を広げて再生成してもらう（推奨）**
   - 同じ会話で以下を追加指示:
   - 「Please regenerate the same sprite sheet, but with MORE SPACING between each sprite. Each sprite must be completely separated with at least 30 pixels of green background between them. No sprite should overlap or touch another sprite.」
   - 全キャラ分やり直す必要がある

B) **手動で各フレームの切り出し座標を指定する**
   - player_frames_v2.pngを見ながら、各フレームの正確な矩形範囲をピクセル単位で指定
   - 重なっている部分は完全には分離できない限界がある

### 問題2: フレーム番号とアクションの対応が不正確

**症状:** パンチを押しているのにキックっぽいフレームが出る等。

**原因:** 41フレーム抽出したが、各フレームが何のアクションかを目視確認が不十分なまま番号を割り当てた。

**解決策:**
- player_frames_v2.png（番号付きレビュー画像）を見ながら、人間が各フレームの正しいアクションを確認
- 確認結果をアニメーション定義（PA定数）に反映

**現在のマッピング（v8b）:**
```javascript
const PA = {
  idle:[0],
  walk:[1,2,3,4,3,2],
  punch:[11,12,15,22],
  kick:[17,18,20],
  guard:[33],
  guard_hit:[34],
  special:[33,34,36,37,38],
  hurt:[39],
  down:[40],
  jump_up:[7,8],
  jump_kick:[9],
  land:[7],
  run:[5,6]
};
```

### 問題3: 敵キャラも同様のフレーム割り当て問題がある可能性

**症状:** 未確認だが、ダルライザーと同じ処理をしているため同様の問題が発生する可能性が高い。

**解決策:** 各敵キャラについてもフレームレビュー画像を生成して確認する。

### 問題4: 影がキャラから離れて浮いて見える場合がある

**症状:** v6で修正したが、v8で再発の可能性。

**原因:** 影のY座標計算がスプライトの描画サイズ（dh）ではなく当たり判定（h）に基づいている。キャラのscale値やスプライトの余白次第でズレる。

**解決策:** 影のY座標を `P.y + P.h` で描画。各キャラで確認が必要。

## 技術構成

### アーキテクチャ
- 単一HTMLファイル（全スプライトはBase64エンコードで埋め込み）
- Canvas 640×360、CSS `image-rendering: pixelated`で最大800px表示
- フォント: Google Fonts DotGothic16（OFLライセンス、商用利用可）
- 60fps requestAnimationFrame ループ

### スプライト描画の仕組み
```
1. スプライトシート画像（PNG）をBase64でHTML内に埋め込み
2. 各キャラのセルサイズ(cw×ch)とスケール(sc)をSP定数で定義
3. フレーム番号 × セル幅 = X座標 で該当フレームを切り出し
4. Canvas.drawImage() でスケーリングして描画
5. facing=-1の場合はctx.scale(-1,1)で左右反転
```

※つまり「セルサイズの四角い範囲」がそのまま表示される。
フレーム内に余白や隣のスプライトの一部が含まれていると、それも表示される。

### スプライトシート構造
```javascript
const SP = {
  p: {cw:110, ch:140, sc:0.85},  // player (41フレーム)
  n: {cw:75,  ch:120, sc:1.0},   // normal enemy (10フレーム)
  l: {cw:120, ch:140, sc:0.95},  // large enemy (22フレーム)
  t: {cw:90,  ch:110, sc:0.95},  // thin+small enemy (27フレーム)
  b: {cw:110, ch:140, sc:1.0},   // boss (10フレーム)
};
```

### アニメーション定義

## スプライト生成方法（Gemini）

### 基本手順
1. Geminiに参考画像（ダブルドラゴンのSteveスプライトシート等）とキャラ写真を添付
2. プロンプトで「2-3行×12-15ポーズ」を指定
3. 背景はグリーンバック `#00FF00` で指定
4. **スプライト間の間隔を30px以上空けるよう指示**（←重要！）
5. 生成画像をClaudeに渡してグリーン除去→スプライトシート化→ゲーム組み込み

### Geminiプロンプトのコツ
- 「Using the attached XXX as a STYLE and POSE REFERENCE」でスタイル統一
- 各フレームの動作を具体的に記述
- サイズは「approximately 200 pixels tall」で高解像度に
- **「at least 30 pixels of green background between each sprite」を必ず入れる**
- 新しいキャラは新しいGemini会話で作る（混同防止）
- 歩きは「right foot forward」「left foot forward」を明示的に指定
- ガードはKEYSIペンサドールの参考写真を添付する

### グリーン除去の注意点
- Geminiは純粋な#00FF00ではなく、薄い緑(174,253,146等)を背景にすることがある
- 背景色はコーナーピクセルから自動取得して除去する方式が安定
- 除去閾値: `abs(r-bg_r)<25 && abs(g-bg_g)<25 && abs(b-bg_b)<25`
- 追加で `g > 120 && g > r * 1.4 && g > b * 1.4` で緑優勢ピクセルも除去

### 下半身反転について
- 下半身だけを水平反転して「逆足」を作る手法は不自然になりやすい
- Geminiに「left foot forward」「right foot forward」を個別生成させる方が良い
- ウォークサイクルはping-pong方式（1→2→3→4→3→2）が安全
