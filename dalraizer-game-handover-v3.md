## 1. プロジェクト概要

ダブルドラゴン（SFC版）風のベルトスクロールアクションゲーム。ブラウザ上で動作する**単一HTMLファイル**に、全素材（スプライト画像・BGM・効果音・背景画像）をbase64エンコードして埋め込んでいる。

### ゲームの流れ

```
タイトル画面（Title.png + Title.mp3）
  ↓ PRESS START
ステージ1「白河の街」（Stage 1.png + Stage 1.mp3）→ ポンチョ軍団15人
  ↓ クリア
ステージ2「小峰城跡」（Stage 2.png + Stage 2.mp3）→ ポンチョ軍団20人
  ↓ クリア
ステージ3「ダイスの拠点」（Stage 3.png + Stage 3.mp3）→ ポンチョ軍団10人 → ボス:DICE
  ↓ クリア
エンディング（Clear.png + Clear.mp3 + CONGRATULATIONS!）
```

## 3. 技術仕様

### キャンバス
- サイズ: **640×360**（16:9）
- レンダリング: `image-rendering: pixelated`（レトロ風ドット感）
- フォント: DotGothic16（Google Fonts）

### スプライトシステム

全キャラのスプライトシートを`SPR`オブジェクトにbase64で格納。`im`オブジェクトに`Image`として変換済み。

```javascript
// スプライト定義（line 27）
const SP = {
  p: {cw:218, ch:301, sc:0.45},  // ダルライザー（セル幅218, 高301, 倍率0.45）
  n: {cw:267, ch:277, sc:0.35},  // 敵A（大型）
  l: {cw:303, ch:257, sc:0.50},  // 敵B（標準）
  t: {cw:255, ch:255, sc:0.30},  // 敵C（小柄）
  b: {cw:196, ch:262, sc:0.45}   // ボスDICE
};
```

- `cw`/`ch`: スプライトシート上の1コマの幅/高さ（ピクセル）
- `sc`: 描画時の縮小倍率
- 描画サイズ = `cw × sc` × `ch × sc`
- 描画関数: `ds(type, x, y, frame, faceDir, alpha)`
  - `faceDir`: 1=右向き、-1=左向き（canvasのscale(-1,1)で反転）

### アニメーション定義

```javascript
// ダルライザー（line 31）
```

### 敵キャラタイプ（ETテーブル、line 197）

| コード | タイプ | HP | 速度 | 攻撃力 | スプライト | 説明 |
|--------|--------|-----|------|--------|-----------|------|

| l | 大型 | 30 | 1.0 | 12 | l (LA) | 紫ポンチョ太め |
| s | 小柄 | 10 | 2.5 | 4 | t (TA) | 紫ポンチョ小柄 |
| t | 中型 | 18 | 2.0 | 8 | t (TA) | 紫ポンチョ中間 |
| b | ボス | 100 | 1.3 | 15 | b (BA) | DICE |

### 敵AI（有限状態マシン）

```
idle → chase（プレイヤー接近） → attack（攻撃範囲内）→ retreat（攻撃後後退）→ chase（再追跡）
                                     ↑
circle（回り込み）←─ chase（攻撃者多すぎ → 待機）
```

- 同時攻撃制限あり（2体まで）
- ボスDICEは`smash`（必殺技）を持つ
- ボスDICE専用：プレイヤーが負けたら`victory`ポーズ（フレーム23）

### 音声システム

- **BGM**: HTML5 Audio要素（`new Audio()`）でbase64を直接再生。ループ再生。
- **SFX**: Web Audio API（`AudioContext.decodeAudioData`）で再生。
- **ブラウザ自動再生対策**: 初回ユーザー操作（click/keydown/touchstart）で`initAudio()`を呼び、タイトル画面なら`playBGM(0)`も実行。

### 背景システム（最新修正済み）

- タイトル/エンディング: `BG_IMAGES.title` / `BG_IMAGES.clear` をそのまま全画面描画
- ステージ1〜3: `BG_IMAGES.stage_1/2/3` のPNG画像を使用
  - パララックス係数: `sx * 0.15`（プレイヤー移動の15%速度でゆっくりスクロール）
  - 画像が画面幅より短い場合は横に繰り返し（タイリング）
  - 画像読み込み中はグラデーションをフォールバック表示

### 操作方法

| PC | スマホ | アクション |
|----|--------|-----------|
| ←→↑↓ / WASD | D-pad | 移動（上下にも移動可能） |
| Z / J | パンチボタン | パンチ |
| X / K | キックボタン | キック（ランダムでジャンプキック） |
| V / Shift | ガードボタン | ガード |
| C / L | 必殺ボタン | 必殺技（ゲージMAX時） |
| Enter / Space | タップ | スタート |

## 6. スプライトシートの規則

全キャラ共通のルール（TAKEAKIさんが作成）:

1. **起点**: 全アクションの最初のフレームはwalk（待機）ポーズ
2. **配列**: 左から右に順番、3行構成
3. **セルサイズ**: キャラごとに統一（40px間隔のスペース付き）
4. **背景**: 透過PNG

### ダルライザーのスプライト構成

**ファイル1: `Dharuriser_walk_kick_punch.png`**（2800×1020）
- Row 0（0-4）: walk 5フレーム
- Row 1（5-10）: kick 6フレーム（起点はwalk左）
- Row 2（11-15）: special（必殺技）5フレーム ※punchではなくspecial

**ファイル2: `Dharuriser_punch02_kick02_down.png`**（2800×1020）
- Row 0（14-15）: punch 2フレーム（起点はwalk左 → 省略）
- Row 1（16）: jump_kick 1フレーム（kick2、ランダム分岐）
- Row 2（17-18）: down 2フレーム（倒れる→起き上がる、起き上がり中に無敵）

### 敵の死亡フレーム指定（TAKEAKIさん指定）

| キャラ | 死亡時フレーム | 備考 |
|--------|---------------|------|
| enemyA (n) | down左から4番目 | dead:[13] |
| enemyB (l) | down左から4番目 | dead:[15] |
| enemyC (t) | down左から3番目 | dead:[13] |
| DICE (b) | down左から4番目 | dead:[15] |

### DICEの特殊モーション
- **Smash（必殺技）**: down_Smash.pngの2行目（5行目相当）= BA.smash:[19,20,21,22]
- **Victory（勝利）**: プレイヤーが負けた場合、最後のポーズ = BA.victory:[23]（ステージ3限定）

## 10. 修正時のヒント

### ファイルが大きすぎて開けない場合
v10.htmlは約44MB（base64データが大部分）。テキストエディタで開く場合は行番号ジャンプ機能を使うこと。base64データは15〜26行目あたりに集中している。

### スプライトを差し替えたい場合
1. 新しいPNG画像をbase64に変換
2. `SPR`オブジェクト内の該当キーの値を差し替え
3. `SP`のcw/ch/scを新しい画像に合わせて調整
4. `PA/NA/LA/TA/BA`のフレーム番号を新しいシートに合わせて変更

### 背景を差し替えたい場合
1. 新しいPNG画像をbase64に変換（`data:image/jpeg;base64,...`形式）
2. `BG_IMAGES`オブジェクト内の該当キー（stage_1等）を差し替え

### 音楽を差し替えたい場合
1. MP3/WAVファイルをbase64に変換
2. `AUDIO_*`定数の値を差し替え
